Git 高频实用技巧 ✨

1. 查看简洁的提交日志
git log --oneline --graph --all
一行展示所有提交，还能看到分支结构，非常清晰。
2. 撤销最近一次提交（不删除代码）
git reset --soft HEAD^
撤回 commit，但代码保留，可重新修改后提交。
3. 丢弃所有未提交的修改（危险！慎用）
git checkout .
恢复到最近一次提交状态，未保存的修改会消失。
4. 暂存当前工作（不想提交但要切分支）
git stash
把代码临时藏起来，恢复用 git stash pop。
5. 强制覆盖本地代码（与远程完全一致）
git fetch --all && git reset --hard origin/主分支名
本地乱了，直接一键同步远程最新版。
6. 修改最后一次提交信息
git commit --amend
刚提交写错备注，用这个直接修改。
7. 查看谁写了哪行代码
git blame 文件名
查BUG神器，看每一行是谁写的。
8. 删除本地已经合并过的分支
git fetch --prune
清理本地无效分支，保持干净。
9. 放弃某个文件的修改
git checkout -- 文件名
只恢复单个文件，不影响其他代码。
10. 把其他分支的某个提交合并过来
git cherry-pick 提交ID
只拿需要的提交，非常灵活。
